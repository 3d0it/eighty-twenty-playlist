# Contributing to EightyTwentyPlaylist

Thank you for considering contributing to EightyTwentyPlaylist! We welcome all kinds of contributions, including bug reports, feature requests, documentation improvements, and code changes.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Open an issue** to discuss your proposed change before making it (optional but recommended).
3. **Write clear, concise commit messages** and include tests for new features or bug fixes.
4. **Ensure your code follows the project's style and conventions.**
5. **Submit a pull request** and describe your changes in detail.

## Code Style
- Use .NET 10 conventions and C# 14.0 features.
- Use XML documentation for all public classes and methods.
- Keep methods short and focused.
- Write unit tests for all new logic.

## Running Tests
- Run `dotnet test` in the solution directory to execute all tests.

## Reporting Issues
- Please use the [GitHub Issues](../../issues) page to report bugs or request features.
- Include as much detail as possible (steps to reproduce, expected behavior, screenshots, etc.).

## Code of Conduct
- Be respectful and inclusive in all interactions.
- See [Contributor Covenant](https://www.contributor-covenant.org/) for general guidelines.

Thank you for helping make EightyTwentyPlaylist better!
